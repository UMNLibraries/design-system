Browser support

Adapted from ClearLeft's browser support policy.

Features, not browsers

Browser support is based on capabilities of the browsers rather than their name or version. The Baseline initiative provides clear information about which web standards features are ready to use in websites. It designates new features into two categories:

  • Newly available: a feature is supported by the latest versions of all core browsers
  • Widely available: a feature has been supported across browsers for at least 30 months

A feature must be "widely available" to be used. A "newly available" feature may be used if it can be implemented as progressive enhancement.

Progressive enhancement means missing the feature does not harm the browser or prevent access to mission-critical requirements or content. Implementing "newly available" features must be nice-to-have additions with usable degradation.

Support and optimization

We support every browser and optimise for none.

Support means that the core content and functionality of a website should be available to any browser. That doesn't mean the content will look exactly the same in every browser.

It may also mean that every interaction may not be identical; some may be implemented as progressive enhancements in newer browsers provided the interaction isn't part of the core functionality of the website.

Avoid polyfills, hacks, and workarounds that would need later removal. Follow Interop to keep abreast of features browser makers have collectively agreed to prioritize.

Two foundational principles underly this browser support policy:

  • Website content and core functionality must be accessible to everyone.
  • It's okay for websites to look different in different browsers.

Content displaying slightly differently in some browsers is a facet of the web, not a bug. This means there will sometimes be subtle visual and functional differences from browser to browser, which is acceptable provided content and core functionality are unaffected.
